## 二、核心策略：双版本 + 小红书私域交付

```
免费版（v1.0）              付费版（v2.0+）
├─ 公开 GitHub Pages        ├─ 私有部署（不可公开访问）
├─ 永久免费                   ├─ 付费后获取
├─ 不再更新                   ├─ 持续迭代
└─ 作为引流入口               └─ 座位轮换、小组模式、批量换座等新功能
```

### 为什么是"双版本"而不是"应用内解锁码"？

纯前端 HTML 无后端，任何锁都可以被绕过（F12 改代码即可）。与其做形同虚设的锁，不如**直接物理隔离**——免费版和付费版是两个不同的部署地址，付费地址不公开。

## 三、付费版交付方式

### 方案 A：GitHub Private Repo + GitHub Pages（推荐）

| 步骤 | 操作 |
|------|------|
| ① 代码管理 | 创建一个 **private** GitHub 仓库存放付费版代码 |
| ② 部署 | Private repo 也能开启 GitHub Pages，但地址**不可被搜索引擎索引** |
| ③ 交付 | 用户付费后，私信发送付费版专属 URL |
| ④ 更新 | 你 push 代码后自动部署，付费用户无感升级 |

**优点**：零成本，自动化部署，和免费版同一套流程
**缺点**：URL 可能被分享（但无伤大雅——用户群小、靠信任运营）

### 方案 A-Plus：加一层轻度访问控制（可选）

在付费版 `index.html` 顶部加一段简单校验：

```javascript
// 付费版入口校验（轻量，非安全锁，仅为防随手传播）
const AUTH = localStorage.getItem('seatplan_paid');
if (!AUTH) {
  const code = prompt('请输入授权码（通过小红书私信获取）：');
  if (code === 'TEACHER2024_SEATPLAN') {  // 定期更换
    localStorage.setItem('seatplan_paid', '1');
  } else {
    document.body.innerHTML = '<div style="text-align:center;padding:100px;"><h2>🔒 付费版本</h2><p>请在小红书 @你的账号 私信获取授权码</p></div>';
  }
}
```

本质是"礼貌性提示"，防君子不防小人，但配合 URL 不做公开分享已经足够。

## 八、常见风险与应对

| 风险 | 应对 |
|------|------|
| 付费版 URL 被公开分享 | 定期更换授权码；小红书私域传播天然限制扩散范围 |
| 用户破解纯前端代码 | 核心价值在持续更新和服务，不在代码加密 |
| 付费用户少 | 免费版本身就是最好的广告，先积累口碑再谈转化 |
| 小红书账号被封 | 多平台分发（抖音/公众号同步运营）作为备选 |
